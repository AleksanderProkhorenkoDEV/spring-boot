# Auth Service

A Spring Boot microservice that provides JWT-based authentication. It exposes two endpoints, register and login, and persists user data in a MySQL database. Incoming requests are validated through a custom security filter chain before reaching any protected route.

## Package Descriptions

### config

Contains the three core pieces of the security setup.

`JwtAuthFilter` is a servlet filter that intercepts every incoming request, extracts the JWT from the `Authorization` header, validates it using `JwtService`, and if valid, sets the authenticated principal in the Spring Security context so the request can proceed.

`JwtProperties` maps the JWT configuration values defined in `application.yml` (such as the secret key and expiration time) into a typed bean, keeping all token settings in one place.

`SecurityConfig` defines the Spring Security filter chain. It marks the `/auth/**` and `/public/**` routes as public and requires a valid JWT for everything else. It also registers `JwtAuthFilter` in the chain before the default username/password filter.

### controller

`AuthController` handles the two public endpoints: `POST /auth/register` and `POST /auth/login`. It delegates the actual logic to the service layer and returns the JWT token in the response.

`TestController` exposes a private route used to verify that the JWT filter is working correctly. It requires a valid token to be present in the request — if not, Spring Security returns a `403 Forbidden`. It has no real business logic; its only purpose is to confirm that authenticated requests are not rejected by the security filter chain.

### dto

Contains the request body classes used to deserialize incoming JSON payloads.

`RegisterRequest` holds the fields required to create a new user (for example, username, email, and password).

`LoginRequest` holds the credentials submitted during a login attempt (for example, username and password).

### entities

Contains the JPA entity classes that map to the MySQL database tables, including the user entity that stores credentials and roles.

### repositories

Contains the Spring Data JPA repository interfaces used to query and persist user data in MySQL. Typically includes a method to look up a user by username or email, used during login and token validation.

### services

`JwtService` handles all JWT operations: generating a token after a successful login or registration, extracting claims from an existing token, and validating whether a token is still valid and untampered.

`UserDetailsServiceImpl` implements Spring Security's `UserDetailsService` interface. It loads the user from the MySQL database by username, maps it to a `UserDetails` object, and is also responsible for saving new users during registration (hashing the password before persisting).

## Authentication Flow

### Register

1. Client sends `POST /auth/register` with a `RegisterRequest` body.
2. `AuthController` delegates to `UserDetailsServiceImpl`.
3. The password is hashed and the user is saved to MySQL.
4. A JWT is generated by `JwtService` and returned to the client.

### Login

1. Client sends `POST /auth/login` with a `LoginRequest` body.
2. `AuthController` verifies the credentials against the database.
3. If valid, `JwtService` generates a JWT and returns it to the client.

### Authenticated Requests

1. Client includes the token in the `Authorization: Bearer <token>` header.
2. `JwtAuthFilter` intercepts the request, validates the token, and sets the security context.
3. The request proceeds to the protected endpoint.

| Method | Path           | Access  | Description                             |

## Database

This service connects to a **MySQL** database to persist user data. Configure the connection in `src/main/resources/application.properties`:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/auth_db
spring.datasource.username=your_username
spring.datasource.password=your_password
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

jwt.secret=your_secret_key
jwt.expiration=86400000
```
